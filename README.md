# AmpUp Skill
#### Video Demo: [Click HERE](https://www.youtube.com/watch?v=lQCrZXJB3DU)
#### Description:

AmpUp Skill is an open-source course platform built with **Flask** and **SQLite** that curates free YouTube content into structured career tracks. Inspired by platforms like Udemy and Coursera, AmpUp Skill organizes high-quality, freely available video lectures from YouTube into phase-based curricula across four in-demand career paths: **Blockchain Development**, **Backend Engineering**, **DevOps Engineering**, and **Machine Learning**.

The platform allows users to register, log in, and track their progress as they work through each track's lectures. Each career track is broken down into sequential phases (e.g., "Phase 1: Blockchain Fundamentals", "Phase 2: Solidity & Smart Contracts") with curated lectures under each phase. Users can mark lectures as complete, and their progress is persisted in the database and visualized on a personal dashboard with circular progress rings.

## Why I Built This

I wanted to solve a real problem I experienced as a self-taught learner — **the overwhelming number of free resources on YouTube with no clear structure**. Platforms like Udemy charge for courses that are sometimes no better than what's freely available. AmpUp Skill bridges this gap by giving learners a structured, phase-by-phase curriculum using the best free content that already exists on YouTube, from creators like Patrick Collins (Blockchain), TechWorld with Nana (DevOps), 3Blue1Brown (ML), and freeCodeCamp.

## Features

- **4 Career Tracks** — Blockchain, Backend Engineering, DevOps, Machine Learning, each with 4 phases and 14-16 curated YouTube lectures
- **User Authentication** — Register and log in with securely hashed passwords (using Werkzeug's `generate_password_hash` / `check_password_hash`)
- **Progress Tracking** — Mark lectures as complete/incomplete with a single click (AJAX-powered, no page reload)
- **User Dashboard** — Circular SVG progress rings per track, "Continue Learning" links, and a recent activity feed
- **Roadmap Integration** — Each track links to its corresponding roadmap on [roadmap.sh](https://roadmap.sh) for a visual overview of the learning path
- **Embedded YouTube Player** — Watch lectures directly on the platform with a responsive 16:9 iframe embed
- **Resource Links** — Each lecture includes additional resource links (documentation, tools, articles)
- **Previous/Next Navigation** — Navigate between lectures sequentially across phases
- **Responsive Design** — Mobile-friendly dark-theme UI with glassmorphism, gradient accents, and CSS animations
- **Flash Messages** — Auto-dismissing success/error notifications for login, registration, and form validation

## File-by-File Explanation

### `app.py`

This is the core of the application containing all the Flask route definitions and configuration. The app uses **server-side sessions** via `Flask-Session` (stored on the filesystem) to persist login state, and `sqlite3` with `Row` factory for database access. A `get_db()` helper function (attached to Flask's `g` context) ensures a single database connection per request that is automatically closed on teardown.

Key routes include:
- `/` — Queries all tracks from the database and renders the homepage with four career track cards.
- `/track/<slug>` — Fetches a track by its URL slug, then retrieves all associated phases and lectures. If a user is logged in, it also checks the `progress` table to determine which lectures have been completed and calculates an overall completion percentage.
- `/track/<slug>/lecture/<id>` — Renders the video player page with the YouTube embed. It queries neighboring lectures to enable previous/next navigation, and parses the JSON `resource_links` column to display additional learning resources.
- `/register` and `/login` — Handle form submissions with input validation, password hashing, duplicate username checks, and appropriate flash messages.
- `/toggle-complete` — A `POST`-only AJAX endpoint that inserts or deletes a row in the `progress` table and returns JSON, allowing the frontend to toggle lecture completion without a full page reload.
- `/dashboard` — Aggregates progress data across all tracks for the logged-in user, calculates per-track completion percentages, identifies the next uncompleted lecture in each track, and fetches the 10 most recent completions.

### `helpers.py`

Contains two utility functions:
- `login_required(f)` — A decorator that redirects unauthenticated users to `/login`. This follows the standard CS50 pattern using `functools.wraps` to preserve the original function's metadata.
- `get_youtube_id(url)` — Extracts the 11-character video ID from various YouTube URL formats (watch, short, embed) using regex pattern matching. This is used during seeding to populate the `youtube_id` column.

### `schema.sql`

Defines five tables with foreign key relationships:
- `users` — Stores usernames and password hashes, with a `created_at` timestamp.
- `tracks` — Stores the four career tracks with metadata (slug for URL routing, name, description, icon emoji, accent color hex code, and roadmap URL).
- `phases` — Each phase belongs to a track (via `track_id` foreign key) and has an ordering number.
- `lectures` — Each lecture belongs to a phase and stores the YouTube URL, extracted video ID, duration string, and a JSON column for additional resource links.
- `progress` — A junction table linking users to completed lectures with a `UNIQUE(user_id, lecture_id)` constraint to prevent duplicate entries.

### `seed.py`

A standalone script that drops the existing database, recreates all tables from `schema.sql`, and inserts curated content. It populates 4 tracks, 16 phases (4 per track), and 58 lectures total. Each lecture entry includes a real YouTube URL, the extracted video ID, an estimated duration, and JSON-encoded resource links pointing to official documentation or learning tools. Running `python seed.py` regenerates the entire database from scratch.

### `static/style.css`

A comprehensive CSS design system (~900 lines) implementing a dark-theme UI. It uses CSS custom properties (variables) extensively for colors, spacing, and effects, making the theme easy to modify. Key design elements include:
- Glassmorphism navbar with `backdrop-filter: blur()`
- Animated hero section with floating gradient blobs
- Track cards with colored accent glows on hover
- Collapsible phase accordion blocks
- Responsive 16:9 YouTube video container
- Circular SVG progress rings on the dashboard
- `fadeInUp` animations with staggered delays for cards
- Full mobile responsiveness via CSS Grid and media queries

### `static/js/main.js`

Handles three client-side interactions:
- **Mobile navigation toggle** — Shows/hides the nav links on small screens and swaps the hamburger/close icon.
- **Phase accordion** — The `togglePhase()` function toggles CSS classes to expand/collapse phase lecture lists.
- **AJAX completion toggle** — `toggleComplete()` sends a `POST` request to `/toggle-complete` and updates the UI (checkbox icon, button text, CSS classes) based on the JSON response, without reloading the page.

### Templates

All templates extend `layout.html` using Jinja2 template inheritance (`{% extends %}` / `{% block %}`). The layout provides the HTML boilerplate, Google Fonts import (Inter), Font Awesome icons, the responsive navbar (which conditionally shows login/signup or username/logout based on session state), flash message rendering with auto-dismiss, and a footer.

The `track.html` template is the most complex — it iterates over phases and their nested lectures, conditionally rendering either completion checkboxes (for logged-in users) or simple number badges. The `dashboard.html` template uses inline SVG with calculated `stroke-dasharray` values to render circular progress rings.

## How to Run

```bash
# Install dependencies
pip install -r requirements.txt

# Seed the database with course content
python seed.py

# Start the development server
flask run --debug
```

Then open [http://127.0.0.1:5000](http://127.0.0.1:5000) in your browser.
